리스트라는 말은 어떤것이 하나가 아닌 여러개가 쭉 일렬로 나열되어있을때 그것을 리스트라고 부릅니다.
덧붙여서 리스트List와 집합Set의 차이를 생각해볼수있습니다.

집합에는 순서라는 개념이없습니다. 단지 포함되어있다는 사실.

+ 랜덤 엑세스?  일렬로 나열된 데이터에 랜덤으로 접근할 때 걸리는 시간이 차이가 없을 때 그 방법.


* 리스트
	- 기본적인 연산 : 삽입, 삭제, 검색 등
	- 리스트를 구현하는 대표적인 두 가지 방법: 배열, 연결리스트.

* 배열의 단점
	- 크기가 고정 - reallocation이 필요.
	- 리스트의 중간에 원소를 삽입하거나 삭제할 경우 다수의 데이터를 옮겨야.

* 연결리스트
	- 다른 데이터의 이동없이 중간에 삽입이나 삭제가 가능하며,
	- 길이의 제한이 없음.
	- 하지만 랜덤 엑세스가 불가능. (배열은 가능).
	- 각 노드들로 묶여있는 자료구조.

* 노드
	- 각각의 노드는 필요한 데이터 필드와 하나 혹은 그 이상의 링크 필드로 구성.
	- 링크 필드는 다음 노드의 주소를 저장.
	- 첫 번째 노드의 주소는 따로 저장.


연결 리스트에서 하나의 노드를 표현하기 위해 구조체이다.
각 노드에 저장될 데이터는 하나의 문자열이라고 가정하자.

	struct node {
		char *data;  // 데이터 필드.
		struct node *next;  // 링크 필드.
	}
	typedef struct node Node;
	Node * head = NULL;

연결 리스트에서 첫번째 노드의 주소를 저장해야만한다.
